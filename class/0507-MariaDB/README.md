# MariaDB 쿼리 연습

### 조인
- equi join : ```=(equal)``` 연산자를 사용하는 조인
  - 조인을 위한 테이블이 N개라고 하면 조인을 위한 최소한의 조건은 N-1 이다.
  - from 절에 테이블을 쭉 나열하고, where 절에 조인 조건을 작성한다.
  - 사용되는 절이 다르다는 것 말고는 이너 조인과 같다.
- inner join : 조인 조건을 만족하는 행에 대해서만 결과값이 나오는 조인
  - 조인을 위한 테이블이 N개라고 하면 조인을 위한 최소한의 조건은 N-1 이다.
  - 조인 절을 사용하고, ON 에 조인 조건을 작성한다.
- outter join : 조인 조건을 만족하지 않아도 출력이 가능해야하는 결과를 얻고자 할 때 사용
  - left, right, full이 있다. (full 조인은 mysql/mariadb 에서 지원 안함)
- 잘 사용되지 않는 join
  - natural join : 각 테이블의 컬럼명이 같을 경우 on 조건을 생략한다.
  - join~using : natural join시 이름이 같은 컬럼이 많은 경우 특정 컬럼과 조인하고 싶은 컬럼에 대해 using절을 사용한다.

### 정규화
- cdmall.exerd 파일 참고
- 1정규화 : 하나의 엔티티가 모든 속성들이 하나의 값을 가질 때
  - 하나의 컬럼에 하나의 데이터씩
  - 비즈니스와 무관한 ID를 PK로 둔다.
  - 일대일, 일대다 관계가 되도록 한다.
  - 다대다 관계는 중간 테이블을 넣어 일대다 관계로 풀어낸다.
- 2정규화 : PK를 제외한 나머지 컬럼들이 엔티티의 PK에 종속적 이어야 한다.
  - 가수 이름, 배급사 번호가 앨범 번호에 종속적이지 않다.
- 3정규화 : PK 제외한 나머지 속성들이 PK가 아닌 다른 속성에 종속적인 경우 이를 해결
  - 장르이름, 장르약어가 있는 경우 장르약어는 장르이름에 종속적이다.
- 해당 정규화 과정이 일률적이지 않고 순서적이지 않다.
- 정확한 과정을 따르는 것은 중요하지 않다.
